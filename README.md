# DBMS---Project-1
Product Safety & Regulatory Compliance System

## Overview
This project is a Database Management System (DBMS) focused on product safety and regulatory compliance. It is designed as a question-answering system that allows users to interact with product data using natural language.

The system stores information about products, inspections, and certification status, helping users easily understand compliance and safety patterns.

---
## Project Objective
- Build a system to manage product safety data  
- Enable natural language queries on compliance data  
- Improve accessibility and understanding of regulatory information  
- Combine structured databases with intelligent retrieval methods  

---

## Project Track
**Track A: Retrieval-Augmented Generation (RAG) Pipeline**

---

## Features
- Store product information (category, expiry date, certification)  
- Track inspection and compliance status  
- Query data using natural language  
- Retrieve accurate, data-driven answers  
- Analyze safety and compliance patterns  

---

## Database Design

### Entities
- **Products**
- **Inspections**
- **Certifications**
- **Compliance Records**

### Relationships
- A product can have multiple inspections  
- A product can have certification status  
- Compliance records are linked to products  

---

## Data Source
- Data collected from real-world observation in:
  - Supermarkets  
  - Pharmacies  
- Includes:
  - Product category  
  - Expiry date  
  - Certification labels  
- Expanded using structured simulation to ensure realistic datasets
